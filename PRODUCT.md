# Product

## Register

brand

## Users

Full-stack developers working with Node.js backends who need architecture discipline and automated enforcement of architectural rules. They manage growing codebases and want to prevent structural decay without sacrificing development speed.

## Product Purpose

Forge is a backend architecture OS that models, builds, audits, and evolves backend systems. It provides 4 layers, 10 commands, and 9 rules to impose automated architectural discipline. Success means developers ship well-structured, auditable, maintainable backend systems with clear ownership and dependency graphs.

## Brand Personality

**Precise, Technical, Powerful.** Voice is confident and direct — no marketing fluff, no filler copy. Interface communicates competence through clarity, tight typography, and zero visual waste. The tool feels like precision engineering: every detail intentional, every interaction purposeful.

## Anti-references

**AI slop aesthetic.** Explicitly reject:
- Cream/sand/paper backgrounds (the saturated warm-neutral AI default)
- Gradient text and decorative glassmorphism
- Rounded cards repeated as the only content pattern
- Tracked uppercase eyebrows above every section
- Numbered section scaffolds (01/02/03) as default decoration
- Generic SaaS landing page structure
- Side-stripe borders as accent elements

## Design Principles

1. **Technical confidence** — Interface speaks like an expert. Code is first-class. Copy is terse and precise, never explanatory padding.
2. **Dark precision** — Dark theme signals technical depth. Orange accent provides energy without warmth.
3. **Zero decorative waste** — Every visual element serves communication. No gradient backgrounds, no decorative blobs, no gratuitous whitespace.
4. **Motion with purpose** — Animations reveal hierarchy and guide attention, never decorate. Respect reduced-motion.
5. **Show the code** — Monospace typography, terminal-inspired aesthetics, inline code blocks as first-class UI elements.

## Accessibility & Inclusion

- WCAG AA compliance (minimum 4.5:1 contrast for body text, 3:1 for large text)
- Full `prefers-reduced-motion` support with instant transitions as alternative
- Semantic HTML structure
- Keyboard navigable with visible focus states
